## 🎨 设计系统概览

### 配色方案 - 群青色系 (Rams Edition)

**主色调**:
- **主色**: 群青蓝 #0437F2 (Ultramarine Blue)
- **深蓝**: #002B8C (标题、重要文字)
- **浅蓝**: #E6F0FF (背景、选中态)

**功能色（克制使用）**:
- **成功色**: 森林绿 #059669 (降低饱和度)
- **警告色**: 琥珀色 #D97706 (降低饱和度)
- **错误色**: 深红 #DC2626 (降低饱和度)

**中性色**:
- **页面背景**: 极浅灰 #FAFAFA
- **卡片背景**: 白色 #FFFFFF
- **边框**: #E5E5E5
- **文字**: #171717 (主要) / #525252 (次要) / #737373 (辅助)

**设计原则**:
- ✅ 主色仅用于主要操作和强调
- ✅ 功能色保留但降低使用频率
- ✅ 移除金色 #F59E0B（过度装饰）
- ✅ 背景使用极浅灰增加阅读舒适度

### 图标系统 - Lucide Icons

**图标库**: lucide-react ^0.400.0
**特点**: 极简线性风格 (2px 线宽)、一致几何美学、完全开源

### 2.3 技术实现要点

#### 表单验证（Zod Schema）
```typescript
const registerSchema = z.object({
  name: z.string().min(2, '姓名至少2个字符'),
  email: z.string().email('请输入有效的邮箱地址'),
  phone: z.string().regex(/^1[3-9]\d{9}$/, '请输入有效的手机号码'),
  password: z.string().min(8, '密码至少8个字符')
    .regex(/[A-Z]/, '需包含大写字母')
    .regex(/[a-z]/, '需包含小写字母')
    .regex(/[0-9]/, '需包含数字'),
  confirmPassword: z.string(),
  field: z.string().optional(),
  agreeTerms: z.boolean().refine(val => val, '请同意用户协议')
}).refine(data => data.password === data.confirmPassword, {
  message: '两次输入的密码不一致',
  path: ['confirmPassword']
});
```

#### API调用
- **POST** `/auth/register`
- 请求体: `{ name, email, phone, password, field? }`
- 响应: `{ success: true, message: '注册成功，请登录' }`

```typescript
interface Section {
  id: string;
  title: string;
  content: string;
  minWords: number;
  maxWords: number;
}

function calculateProgress(section: Section): number {
  const wordCount = section.content.length;
  if (wordCount >= section.minWords) return 100;
  return Math.round((wordCount / section.minWords) * 100);
}
```

## 附录D：Dieter Rams 设计十诫

本设计系统遵循 Dieter Rams 的设计十诫（Ten Principles for Good Design），进行**适度 Rams 化**：

### 1. 创新但不炫技 (Innovative)
**原则**: Good design is innovative
**应用**: 保留 AI 功能亮点，但避免过度动画和装饰效果

### 2. 有用且诚实 (Makes a product useful & Honest)
**原则**: Good design makes a product useful & honest
**应用**:
- 功能明确，不夸大承诺
- 开发中功能明确标注
- 不使用欺骗性视觉元素

### 3. 美学服务功能 (Aesthetic)
**原则**: Good design is aesthetic
**应用**:
- 视觉美感支持操作效率
- 配色协调不刺激
- 布局清晰易导航

### 4. 清晰易懂 (Understandable)
**原则**: Good design makes a product understandable
**应用**:
- 信息层级明确
- 图标语义清晰（Lucide Icons）
- 减少认知负担

### 5. 克制但不冷漠 (Unobtrusive)
**原则**: Good design is unobtrusive
**应用**:
- 保留学术场景的温度感
- 功能色克制使用
- 避免视觉噪音

### 6. 诚实呈现 (Honest)
**原则**: Good design is honest
**应用**:
- 未实现功能明确标注
- 错误提示清晰准确
- 不过度承诺产品能力

### 7. 持久耐看 (Long-lasting)
**原则**: Good design is long-lasting
**应用**:
- 避免流行趋势
- 追求永恒性设计
- 主色群青蓝经典耐看

### 8. 细节完美 (Thorough)
**原则**: Good design is thorough down to the last detail
**应用**:
- 8px 栅格系统（数学之美）
- 4px 统一圆角
- 2px 统一图标线宽

### 9. 环保高效 (Environmentally-friendly)
**原则**: Good design is environmentally-friendly
**应用**:
- 轻量级组件
- Tree-shaking优化
- 性能优先

### 10. 尽可能少 (As little design as possible)
**原则**: Good design is as little design as possible - Less but Better
**应用**:
- 每个元素存在都有理由
- 移除装饰性emoji
- 简化断点系统（6→3）
- 减少阴影效果

### 设计实践总结

**Less but Better（少即是多）**体现在：
1. ✅ 配色从多彩简化为群青主导
2. ✅ 断点从6个简化为3个
3. ✅ 按钮样式从5种简化为3种
4. ✅ 移除装饰性元素（emoji、过度阴影）
5. ✅ 统一视觉语言（4px圆角、8px间距）

**适度 Rams 化**原则：
- ✅ 不走极端：保留必要的功能色
- ✅ 保留温度：不追求极致冷淡风
- ✅ 实用主义：视觉服务于功能
- ✅ 循序渐进：分阶段实施改造
